# Фабрика гипотез

B2B-инструмент для технологов обогатительных фабрик: загружаете Excel-отчёт
института по анализу хвостов — получаете ранжированный список **проверяемых
гипотез** по снижению потерь Ni и Cu, каждая с механизмом, цитатами из
литературы (с указанием страницы), оценкой эффекта в тоннах и деньгах, рисками
и планом проверки. Плюс тепловая карта потерь, детерминированная диагностика,
дорожная карта испытаний (Гант с учётом занятости оборудования), экспорт
DOCX/CSV/JSON и чат-интерпретатор.

Ключевые свойства:
- **интерпретируемость** — каждый диагноз хранит правило (R1–R5), входные числа и порог; показывается в UI по клику;
- **работа с шумными данными** — #REF! и битые ячейки восстанавливаются по балансовым инвариантам (ярус 1, точная формула) или LLM-оценкой с confidence (ярус 2), подсвечиваются и правятся вручную;
- **цитируемость** — каждая цитата верифицируется fuzzy-матчем по чанку источника; невалидные помечаются «требует проверки», гипотезы без подтверждённых цитат понижаются в ранге;
- **локальное развёртывание** — SQLite + chromadb + локальные эмбеддинги (bge-m3), LLM — любой OpenAI-совместимый эндпоинт;
- **масштабируемость на другие домены** — вся доменная логика (синонимы меток, правила, оборудование, коэффициенты эффекта, профили проверки) вынесена в `domain_packs/flotation.yaml`.

## Запуск за 3 команды

Требования: Python 3.11+, Node 18+. Данные кейса положите в `data/case/`
(структура — как в задании: `Пример 1..4`, `Дополнительные материалы`, …).

```bash
# 1. зависимости (requirements-ml.txt опционален: без него KB работает на BM25)
pip install -r requirements.txt -r requirements-ml.txt && cd frontend && npm install && cd ..

# 2. конфиг LLM: скопируйте шаблон и впишите ключ (без ключа всё работает на моках)
cp .env.example .env

# 3. два процесса
python -m uvicorn backend.app.main:app --port 8000     # бэкенд
cd frontend && npm run dev                              # фронтенд -> http://127.0.0.1:5173
```

Наполнить базу для демо (проект «НОФ · вкрапленные руды · Q2 2026», Пример 2,
4 PDF в базе знаний, гипотезы, статусы, дорожная карта):

```bash
python scripts/demo_seed.py          # с ключом — живые вызовы; --mock — без сети
```

Тесты и бенчмарк:

```bash
pytest -q                            # 61 тест, LLM замокана
python eval/run_eval.py              # сравнение с эталонами экспертов -> eval/report.md
```

## Демо-сценарий (10 шагов)

1. **Разбор отчёта.** Главная → проект демо-сида (или создайте и загрузите
   `Пример 2/Хвосты НОФ Вкр.xlsx`). Карточки: 4.38 млн т хвостов, потери
   Ni 4392 т / Cu 1506 т, извлекаемо 60% / 75%.
2. **Шумные данные.** Загрузите `Пример 1/Хвосты КГМК.xlsx` в новый проект:
   парсер не падает — 22 битые ячейки (#REF!) восстановлены по инвариантам,
   подсвечены янтарным пунктиром, в тултипе формула («класс +125 пуст → 0 т»),
   счётчик «Восстановлено N — проверьте», кнопка «Принять все восстановленные».
3. **Карта потерь** (шаг 2 в степпере): тепловая матрица классы×формы,
   неизвлекаемые колонки с 🔒, переключатель Ni/Cu. Справа — диагнозы:
   [R1] недоизмельчение (закрытый Pnt/Cp в крупных классах, 27.7% потерь),
   [R2] переизмельчение (раскрытый в −10). Клик по бейджу правила — входные
   числа и пороги. Ниже — аномалия R5: 56.4% потерь Cu класса −10 в пирите
   («проверьте пробу») и блок «Почему не предложено» с тоннами по
   неизвлекаемым формам.
4. **Генерация гипотез** (шаг 3): кнопка слева (живой вызов ~3–4 мин; без
   ключа — мгновенный мок). Карточки: эффект «до N т/год ≈ $M», скоринг-бар,
   цитаты-чипсы — клик открывает фрагмент книги с подсветкой, бейдж ✓
   verified. Оборудование «✓ есть на фабрике (поз. 5-3)». Зелёный бейдж
   «Совпадает с гипотезой экспертов».
5. **Фидбэк.** «Отклонить» с причиной «футеровка уже меняется по графику» →
   причина уходит в стоп-лист; повторная генерация это направление не предложит.
6. **Чат** (💬 в шапке): «почему гипотеза №1 первая?» → ответ с кликабельными
   ссылками [R1], [+125/Закрытый Pnt/Cp/Ni] — клик ведёт на карту/карточку.
7. **Дорожная карта** (шаг 4 → вкладка): примите 3 гипотезы (две мельничные)
   → «Построить»: Гант со стадиями лаборатория → ОПИ → тираж, ромбы-ворота с
   числовыми критериями, конфликт по мельнице разведён последовательно
   («⏳ ждёт мельницу 5-3»), линия «сегодня». Сдвиг сегмента в занятый слот
   вернёт 409 с причиной.
8. **Экспорт.** Кнопки DOCX (титул, сводка, топ-5, дорожная карта, карточки),
   tasks.csv (start/end/resource/criterion), JSON.
9. **Второй домен.** База знаний → вопрос «как извлекают золото из упорных
   руд?» → ответ с цитатами из статьи по золоту (страница, источник); скан
   Лодейщикова честно помечен «требуется OCR». Домен меняется одним yaml.
10. **Цифры eval** — `eval/report.md`: parse_ok 4/4, coverage и
    citation_validity по прогонам.

## Архитектура

```
xlsx ──► parser/xlsx.py ──► recover.py (7.1: инварианты → LLM) ──► TailingsReport
                                                              │
PDF ──► kb/ingest.py ──► чанки ──► BM25Plus + chromadb(bge-m3) ┐│
                                                              ▼▼
docx (эталоны) ──► eval/run_eval.py     diagnostics.py (R1–R5, без LLM)
                                                              │
                     hypotheses/generate.py (STRONG, JSON) ◄──┘
                     verify.py (rapidfuzz>75) → rank.py (веса+novelty)
                                │
        store.py (SQLite) ◄─────┼────► roadmap.py (Гант, ресурсы)
                                ▼
   FastAPI (backend/app/api.py) ──► React/Vite/Tailwind (6 экранов + чат)
```

- Конфиг LLM — только из `.env` (см. `.env.example`); ключ маскируется во всех логах.
- Без `LLM_API_KEY` весь пайплайн работает, генерация отдаёт мок-фикстуру с цитатами, заземлёнными на локальный индекс.
- Метки классов/минералов нормализуются по числам и словарю синонимов — «+71», « -20 + 10», «Пирит/Другие Элемент 29 сульфиды» парсятся одинаково во всех 4 файлах.
- ТОФ: разбираются обе разновидности хвостов (породные + пирротиновые) и общий блок, Факт/Расчёт — в метаданных.

## API (основное)

| Метод | Путь | Что делает |
|---|---|---|
| POST | `/api/projects` | создать проект |
| POST | `/api/projects/{id}/report` | загрузить xlsx → отчёт с issues и восстановлением |
| PATCH | `/api/projects/{id}/report/cells` | ручная правка ячеек (provenance=manual) |
| GET | `/api/projects/{id}/diagnostics` | диагнозы R1–R5 + карта потерь |
| POST | `/api/projects/{id}/hypotheses/generate` | генерация (веса, ограничения) |
| POST | `/api/hypotheses/{id}/feedback` | accept / reject(+причина → стоп-лист) |
| POST | `/api/projects/{id}/chat` | чат-интерпретатор со ссылками |
| POST | `/api/projects/{id}/roadmap/build` | дорожная карта; PATCH `/api/roadmap/items/{id}` — сдвиг (409 при конфликте) |
| GET | `/api/projects/{id}/export/docx‖tasks.csv‖json` | экспорт |
| POST | `/api/kb/upload`, `/api/kb/ask` | база знаний: инжест PDF, вопрос с цитатами |

Пример curl:

```bash
curl -s -X POST http://127.0.0.1:8000/api/projects -H "Content-Type: application/json" \
  -d '{"plant":"НОФ"}'
curl -s -X POST http://127.0.0.1:8000/api/projects/<id>/report \
  -F "file=@data/case/Пример 2/Хвосты НОФ Вкр.xlsx"
curl -s http://127.0.0.1:8000/api/projects/<id>/diagnostics | jq '.diagnoses[].title'
```
