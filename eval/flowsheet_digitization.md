# Отчёт оцифровки схем флотации и регламентов

Метод: каждое изображение прочитано vision-агентом (извлечение узлов/потоков/значений),
затем НЕЗАВИСИМЫЙ второй агент перечитал ту же картинку и вынес вердикт по каждой цифре
(подтверждено / другое значение / не читается / нет на схеме). В yaml попали только
подтверждённые значения; нечитаемое = null. Результат — `domain_packs/flotation.yaml`,
секция `flowsheets`.

## Инвентаризация (17 файлов)

В оцифровку: Регламент 1, Схемы 3, 4, 5, 6, 7, 8 + 3 листа списка оборудования.
Отсечены как учебные: Схема флотации (Cu-Pb-Zn-пирит), Схема флотации 2 (абстрактная),
Схема флотации 3 (обезличенная), Схема флотации 4 (золото, Нельсон),
схемы флот++.pdf (чертёж Маломырского ЗОЛОТОРУДНОГО рудника).
Не вошли в yaml: Схема 1 (дробление с ЩДП 9×12/КСД 2200 — оборудование вне онтологии
кейса; альтернативная конфигурация, данные сохранены в артефактах воркфлоу),
Схема 2 (аппаратная цепь ИФЦ — числа на схеме отсутствуют, маркировка аппаратов
местами сомнительная: агент прочёл «МШЦ 3,2х3,8», которой не существует).

## Подключение к пайплайну (ШАГ 3)

- Определение фабрики: `detect_factory()` по имени xlsx/названию фабрики (КГМК/НОФ/ТОФ;
  НОФ вкр и мед — одна схема НОФ); оверрайд — селектор на создании проекта; кнопка
  «Загрузить свой регламент» задизейблена с тултипом.
- Диагнозы: `node_refs` + строка «по регламенту: узел, t=…, %тв=…, реагенты=…» в тексте
  (R1 → grinding/classification/crushing, R2/R3 → flotation, контрольные в приоритете).
- Генератор: в промпт уходит выжимка flowsheet (узлы с режимами + хвостовые потоки с β/ε)
  и системные подсказки по реагентам с расходом 0 при наличии KB-источников
  (ДМДК=0 на ТОФ + Манцевич-2008 в корпусе → подсказка с цитатой).
- Экран диагностики: граф переделов из flowsheet фабрики, узлы диагнозов подсвечены.
